  <name>Task 1: Format-preserving DOCX writer with cover letter and ATS checks</name>
  <files>app/tailoring/docx_writer.py</files>
  <action>
Create `app/tailoring/docx_writer.py`:

**Helper: replace_paragraph_text_preserving_format(paragraph, new_text: str):**
- If paragraph has no runs: set paragraph.text = new_text (fallback).
- Otherwise: keep first run's formatting, remove all runs after the first (via `run._element.getparent().remove(run._element)`), set `first_run.text = new_text`.
- CRITICAL: Never use `paragraph.text = ...` when runs exist (per research Pitfall 1).

**Helper: _match_heading(docx_heading: str, json_heading: str) -> bool:**
- Case-insensitive exact match first.
- Then keyword-based fuzzy match: if both contain "experience" or "education" or "skills" or "summary" or "objective" or "projects", match them. This handles "Work Experience" vs "Professional Experience" (per research Pitfall 5).

**Main: build_tailored_docx(base_resume_path: Path, tailored_sections: dict, output_path: Path) -> Path:**
1. Copy base resume to output_path using shutil.copy2 (never modify original).
2. Open the copy with Document().
3. Build a section_map from tailored_sections["sections"], keyed by heading (lowercase).
4. Walk paragraphs:
   - If style starts with "Heading": update current_heading tracker. Do NOT modify heading text (headings are locked).
   - If non-heading paragraph under a matched section:
     - For simple sections (content list): replace paragraphs in order. If tailored has fewer bullets, clear extras. If more, log warning (can't add paragraphs without breaking formatting).
     - For subsections (work experience pattern): match by company name (locked field). Replace only bullet text, preserve company/title/dates paragraphs untouched.
   - If current_heading not in section_map: leave paragraph untouched.
5. If tailored_sections has a "skills" key, find and replace the skills section content.
6. Save document.
7. Return output_path.

**Cover letter: build_cover_letter_docx(paragraphs: list[str], output_path: Path, base_resume_path: Path | None = None) -> Path:**
1. Create a new Document().
2. Try to detect font from base_resume_path if provided (read first run's font.name). Default to "Calibri" if not detectable.
3. Set default font on document style.
4. Add paragraphs with standard business letter spacing.
5. Save to output_path.
6. Return output_path.

**ATS checks: check_ats_friendly(docx_path: Path) -> dict:**
- Open document, check for:
  - `has_tables`: bool -- any Table elements present
  - `non_standard_fonts`: list[str] -- fonts not in {"Arial", "Calibri", "Times New Roman", "Helvetica", "Georgia", "Garamond", "Cambria", "Verdana"}
  - `keyword_coverage`: leave as None (computed separately when job description is available)
- Return dict with these fields.

**Keyword coverage: compute_keyword_coverage(tailored_text: str, job_description: str) -> float:**
- Extract significant words (>3 chars) from job description.
- Count how many appear in tailored text (case-insensitive).
- Return coverage as 0.0-1.0 ratio.

Use structlog for: section matching, replacement counts, ATS check results.
