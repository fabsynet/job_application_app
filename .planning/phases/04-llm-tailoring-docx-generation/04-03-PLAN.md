Create `app/tailoring/prompts.py` with three main prompt constants:

**TAILORING_SYSTEM_PROMPT** (str):
Core instructions for resume tailoring. Must include:
- "You are a professional resume tailoring assistant. Your job is to adapt a base resume for a specific job posting."
- "CRITICAL RULE: You may ONLY use information from the BASE RESUME. You may reword, reorder, and emphasize, but you MUST NEVER invent companies, job titles, skills, technologies, metrics, degrees, or certifications that are not in the original."
- "Reasonable inferences are allowed: if the resume says 'FastAPI', you may say 'Python backend development'. But 'React' is NOT acceptable unless explicitly stated."
- Define three intensity levels with explicit instructions:
  - Light: "Reorder bullet points to lead with most relevant. Emphasize keywords from job description. Minimal rewording."
  - Balanced: "Reorder bullets, rephrase for clarity and impact, adjust framing to match job language. Same facts, different presentation."
  - Full: "Reorder, rephrase, restructure sections. May combine or split bullet points. Maximize relevance while preserving all factual content."
- "LOCKED FIELDS - copy these EXACTLY from the base resume, do not modify: name, email, phone, address, job titles, company names, employment dates."
- Instruct Claude to return JSON matching the TAILORING_OUTPUT_SCHEMA.

**TAILORING_OUTPUT_SCHEMA** (dict, documented in a docstring):
```python
# Expected JSON output structure
{
    "sections": [
        {
            "heading": "Professional Summary",  # exact heading from base resume
            "content": ["Line 1", "Line 2"]     # for simple sections
        },
        {
            "heading": "Work Experience",
            "subsections": [
                {
                    "company": "Acme Corp",      # LOCKED - verbatim
                    "title": "Software Engineer", # LOCKED - verbatim
                    "dates": "2020-2023",         # LOCKED - verbatim
                    "bullets": ["Tailored bullet 1", "Tailored bullet 2"]
                }
            ]
        }
    ],
    "skills": ["Python", "FastAPI"]  # skills section if present
}
```

**VALIDATOR_SYSTEM_PROMPT** (str):
- "You are a strict fact-checker for resume tailoring. Compare the TAILORED resume against the ORIGINAL resume."
- Flag violations: invented companies, titles, skills/technologies not reasonably inferable, metrics/numbers not in original, degrees/certs not in original.
- Define "reasonably inferable" with examples.
- Return JSON: `{"passed": bool, "violations": [{"type": str, "content": str, "explanation": str}]}`

**COVER_LETTER_SYSTEM_PROMPT** (str):
- "Write a professional, concise cover letter (3-4 short paragraphs) for this job application."
- Paragraph 1: Why the candidate is a fit (based on resume + job description)
- Paragraph 2-3: 2-3 specific relevant highlights from the resume
- Paragraph 4: Close with enthusiasm, no fluff
- "Use ONLY facts from the provided resume. Do not invent experiences."
- Return JSON: `{"paragraphs": ["para1", "para2", "para3", "para4"]}`

Create `app/tailoring/engine.py` with the core tailoring orchestration:

**PII stripping helper:**
`strip_pii_sections(sections: list[dict]) -> tuple[str, list[str]]`:
- Takes the sections list from extract_resume_text() output.
- Identify the first section (heading=None or heading containing contact-related words like "contact", "info", "personal"). Skip it -- this typically contains name, email, phone, address.
- Join remaining sections into a single text string for the LLM prompt.
- Return (sanitized_text, list_of_section_headings).
- Log a warning if no sections were stripped (resume may not have a contact header).

**parse_tailoring_response(raw_content: str) -> dict:**
- Parse JSON from Claude's response. Handle markdown code fences (```json ... ```).
- Validate it has a "sections" key that is a list.
- Raise ValueError with descriptive message if parsing fails.

**parse_validation_response(raw_content: str) -> dict:**
- Parse JSON from validator response.
- Must have "passed" (bool) and "violations" (list).
- Raise ValueError if malformed.

**async def validate_output(provider: LLMProvider, original_text: str, tailored_json: str) -> tuple[bool, list[dict]]:**
- Build validator messages via prompts.build_validator_messages.
- Call provider.complete with max_tokens=2048, temperature=0.1.
- Parse response. Return (passed, violations).
- Return the LLMResponse alongside for cost tracking: change signature to return `(bool, list[dict], LLMResponse)`.

**async def generate_cover_letter(provider: LLMProvider, resume_text: str, job_description: str, company: str, title: str) -> tuple[list[str], LLMResponse]:**
- Build cover letter messages.
- Call provider.complete with max_tokens=2048, temperature=0.4.
- Parse JSON response, extract paragraphs list.
- Return (paragraphs, llm_response).

**Main function -- async def tailor_resume(provider: LLMProvider, resume_sections: list[dict], job_description: str, intensity: str = "balanced", max_retries: int = 3) -> TailoringResult:**

Define a `TailoringResult` dataclass at module level:
```python
@dataclass
class TailoringResult:
    success: bool
    tailored_sections: dict | None  # parsed JSON from Claude
    cover_letter_paragraphs: list[str] | None
    validation_passed: bool
    validation_warnings: list[dict]  # all warnings across retries
    total_input_tokens: int
    total_output_tokens: int
    total_cache_read_tokens: int
    total_cache_write_tokens: int
    retry_count: int
    error: str | None
    llm_calls: list[dict]  # per-call cost details for CostLedger
```

Implementation:
1. Call strip_pii_sections to get sanitized text and headings.
2. Build system messages with cache_control (via build_system_messages).
3. **Retry loop (up to max_retries):**
   a. Build user message with intensity + escalation suffix for this retry.
   b. Call provider.complete(system, messages, max_tokens=4096, temperature=0.3).
   c. Record call details (tokens, type="tailor") in llm_calls list.
   d. Parse response JSON.
   e. Call validate_output(). Record validator call details (type="validate").
   f. If validation passed: break out of loop.
   g. If validation failed: log warnings, increment retry, continue loop.
4. If all retries exhausted: set success=False, error="Validation failed after {max_retries} attempts".
5. If validation passed: generate cover letter. Record call details (type="cover_letter").
6. Return TailoringResult with all accumulated data.

Use structlog throughout for: tailoring_start, tailoring_attempt, validation_result, retry, tailoring_complete/failed.
