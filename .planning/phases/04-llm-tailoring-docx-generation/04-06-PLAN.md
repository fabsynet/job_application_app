**Create `app/web/routers/tailoring.py`:**

**GET /tailoring/{job_id}** -- Tailoring detail view:
- Load job by ID.
- Load all TailoringRecord rows for this job via get_tailoring_records_for_job.
- Load latest completed record.
- If latest exists:
  - Generate HTML preview via docx_to_html on the tailored resume.
  - Generate section diff via generate_section_diff + format_diff_html (base resume sections vs tailored sections from the record).
  - Parse validation_warnings JSON string.
  - Load ATS check results via check_ats_friendly.
  - Read keyword_coverage from the TailoringRecord (stored by pipeline in plan 05).
- Render `tailoring_detail.html.j2` with: job, records, latest_record, preview_html, diff_html, validation_warnings, ats_checks, keyword_coverage, cover_letter_preview (if cover letter exists, convert that too).

**GET /tailoring/{job_id}/download/{version}** -- DOCX download:
- Compute path: `resume_artifact_path(job_id, version)`.
- Return FileResponse with media_type "application/vnd.openxmlformats-officedocument.wordprocessingml.document", filename like `tailored_resume_v{version}.docx`.
- 404 if file doesn't exist.

**GET /tailoring/{job_id}/cover-letter/{version}** -- Cover letter download:
- Same pattern, using cover_letter_artifact_path.

**GET /tailoring/{job_id}/preview/{version}** -- HTML preview partial (for HTMX):
- Generate and return HTML preview of a specific version.
- Used for switching between versions in the detail view.

**Create `tailoring_detail.html.j2`:**
- Job info header (title, company, score).
- Version selector (dropdown or tabs if multiple versions).
- Two-column layout:
  - Left: Base resume sections.
  - Right: Tailored resume sections.
  - Changed sections highlighted (use diff_html from format_diff_html).
- Validation status: green check if passed, or list of warnings with retry details.
- **Token usage and cost section** -- must include:
  - Total input/output tokens
  - **cache_read_tokens** and **cache_write_tokens** displayed explicitly (e.g., "Cache read: 1,234 tokens")
  - **Estimated cache savings** calculated as: `cache_read_tokens * (input_price - cache_read_price)` where input_price=3.00/MTok and cache_read_price=0.30/MTok for claude-sonnet-4-5. Display as "Estimated cache savings: $X.XX" (SC-5: prompt caching visibly reduces cost).
  - Total estimated cost for this tailoring.
- Download buttons: "Download Resume" and "Download Cover Letter".
- ATS check summary: tables (yes/no), fonts, **keyword coverage percentage** (from record.keyword_coverage, displayed as "XX% keyword match").

**Create `resume_preview.html.j2`:**
- Simple partial wrapping mammoth HTML output in a styled container.
- Used for both resume and cover letter preview.

**Register router in `app/main.py`:**
Add `from app.web.routers.tailoring import router as tailoring_router` and include it on the app.
