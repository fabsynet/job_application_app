1. Create app/credentials/__init__.py (empty).

2. Create app/credentials/validation.py with two async functions:

   a) `async def validate_anthropic_key(api_key: str) -> tuple[bool, str]`:
      - Use httpx.AsyncClient with timeout=10.0
      - GET https://api.anthropic.com/v1/models with headers {"x-api-key": api_key, "anthropic-version": "2023-06-01"}
      - 200 = (True, "API key is valid")
      - 401 = (False, "Invalid API key")
      - Other status = (False, f"Unexpected response: {status_code}")
      - TimeoutException = (False, "Validation timed out — key saved but not verified")
      - ConnectError = (False, "Could not reach Anthropic API — key saved but not verified")
      - NOTE: On network errors, the key should still be SAVED (save first, validate second). The flash message indicates validation couldn't complete but the key is stored.

   b) `async def validate_smtp_credentials(host: str, port: int, username: str, password: str) -> tuple[bool, str]`:
      - Run in thread pool via asyncio.to_thread
      - Use smtplib.SMTP(host, int(port), timeout=5)
      - server.ehlo(), if port == 587: server.starttls() + server.ehlo(), server.login(username, password)
      - Success = (True, "SMTP credentials valid")
      - SMTPAuthenticationError = (False, "Authentication failed")
      - socket.timeout/TimeoutError = (False, "Connection timed out — credentials saved but not verified")
      - ConnectionRefusedError/OSError = (False, f"Connection failed: {e}")
      - Same principle: save first, then validate. Network failures don't prevent saving.
  </action>

  </verify>
  <done>Validation module provides two async functions: one for Anthropic API key (GET /v1/models, zero-cost), one for SMTP (connection test in thread pool, 5s timeout). Both return (bool, message) tuples.</done>
