**app/discovery/fetchers.py:**

Create ATS fetcher functions following RESEARCH.md patterns exactly:

1. `fetch_greenhouse(client: httpx.AsyncClient, slug: str) -> list[dict]`
   - GET `https://boards-api.greenhouse.io/v1/boards/{slug}/jobs?content=true`
   - Response: `{"jobs": [...]}`
   - Map: id->external_id, title, location.name, content->description (HTML), absolute_url->url, updated_at->posted_date
   - Store content as both description (strip HTML for scoring) and description_html

2. `fetch_lever(client: httpx.AsyncClient, slug: str) -> list[dict]`
   - GET `https://api.lever.co/v0/postings/{slug}?mode=json`
   - Response: flat array `[{...}]` (NOT wrapped in object)
   - Map: id->external_id, text->title, categories.location, descriptionPlain->description, hostedUrl->url
   - posted_date = None (Lever does not expose this)

3. `fetch_ashby(client: httpx.AsyncClient, slug: str) -> list[dict]`
   - GET `https://api.ashbyhq.com/posting-api/job-board/{slug}`
   - Response: `{"jobs": [...]}`
   - Map: id->external_id, title, location, descriptionPlain->description, descriptionHtml->description_html, jobUrl->url, publishedAt->posted_date

4. `detect_source(input_str: str) -> tuple[str, str]`
   - Parse URL patterns: greenhouse.io, lever.co, ashbyhq.com
   - Plain slug (alphanumeric + dash + underscore) returns ("unknown", slug)
   - Raise ValueError for unrecognizable input

5. `validate_source(slug: str, source_type: str) -> bool`
   - Hit the real API with httpx. Return True if valid response, False otherwise
   - For "unknown" source_type, probe greenhouse -> lever -> ashby, return first success as (source_type, slug)
   - Use timeout=10.0 for validation

6. Simple HTML-to-text helper `strip_html(html: str) -> str` using regex to strip tags for scoring purposes. Keep it simple -- just remove `<...>` tags and decode common entities.

**app/discovery/scoring.py:**

1. `job_fingerprint(url: str, title: str, company: str) -> str`
   - Normalize: URL lowercase, strip query/fragment/trailing slash; title lowercase, collapse whitespace; company lowercase, strip
   - SHA256 hash of `"{clean_url}|{clean_title}|{clean_company}"`

2. `score_job(description: str, keywords: list[str]) -> tuple[int, list[str], list[str]]`
   - Case-insensitive partial substring matching per user decision
   - "python" matches "Python", "python3", "Python/Django"
   - Returns (score_0_to_100, matched_keywords, unmatched_keywords)
   - Empty keywords list returns (0, [], [])

CRUD functions using async SQLAlchemy sessions:

13. `get_rolling_average(session, source_id, days=7) -> Optional[float]` - For anomaly detection. Returns None if fewer than 3 data points.

**app/discovery/pipeline.py:**

Create `run_discovery(ctx: RunContext, session_factory) -> dict` that orchestrates:

1. Load enabled sources and user keywords (from Settings.keywords_csv, pipe-delimited)
2. Fetch from all enabled sources in parallel using `asyncio.gather(*tasks, return_exceptions=True)`
   - Use `httpx.AsyncClient(timeout=30.0)` shared across all fetchers
   - Individual source failures are caught and logged, not propagated
   - Update source fetch status (ok/error) after each fetch
3. For each discovered job:
   - Compute fingerprint via `job_fingerprint(url, title, company)`
   - Check dedup: skip if fingerprint already exists in DB
   - Score against keywords via `score_job(description, keywords)`
   - Set status = "matched" if score >= threshold, else "discovered"
   - Persist Job row with score, matched_keywords (pipe-delimited), status
4. Save DiscoveryRunStats per source (discovered_count, matched_count, error)
5. Check anomaly per source: if today's count < 20% of 7-day rolling average AND >= 3 data points, flag anomaly
6. Update Run.counts with {"discovered": N, "new": K, "matched": M} via finalize
7. Return counts dict for logging

Kill-switch checkpoints: call `raise_if_engaged()` before fetching and after persisting.

**app/scheduler/service.py:**

Replace `_execute_stub` with `_execute_pipeline` that:
1. Calls `run_discovery(ctx, self._session_factory)`
2. Updates the Run.counts dict with discovery results
3. Keeps killswitch checkpoints

The method name change from `_execute_stub` to `_execute_pipeline` must be reflected in `run_pipeline()` where it calls `self._execute_stub(ctx)` -> `self._execute_pipeline(ctx)`.

Store anomaly results in the Run.counts dict under an "anomalies" key so the dashboard can read them.
