<task type="auto">
  <name>Task 1: Create discovery models (Source, Job, DiscoveryRunStats)</name>
  <files>app/discovery/__init__.py, app/discovery/models.py</files>
  <action>
Create `app/discovery/__init__.py` (empty).

Create `app/discovery/models.py` with three SQLModel table classes following the schema from RESEARCH.md:

**Source table** (`sources`):
- id: Optional[int], primary key
- slug: str, indexed (e.g. "stripe")
- source_type: str (greenhouse | lever | ashby)
- display_name: Optional[str] (user-friendly label)
- enabled: bool, default True
- last_fetched_at: Optional[datetime]
- last_fetch_status: Optional[str] (ok | error)
- last_error_message: Optional[str]
- created_at: datetime, default utcnow

**Job table** (`jobs`):
- id: Optional[int], primary key
- fingerprint: str, unique=True, indexed (SHA256 dedup key)
- external_id: str (ATS-specific ID)
- title: str
- company: str, indexed
- location: str, default ""
- description: str, default "" (plain text for scoring)
- description_html: str, default "" (HTML for display)
- url: str (original posting URL)
- source: str (greenhouse | lever | ashby)
- source_id: Optional[int], foreign_key="sources.id"
- posted_date: Optional[datetime] (nullable -- Lever lacks this)
- score: int, default 0 (0-100 keyword overlap)
- matched_keywords: str, default "" (pipe-delimited)
- status: str, default "discovered" (discovered | matched | queued | applied)
- first_seen_at: datetime, default utcnow
- run_id: Optional[int], foreign_key="runs.id"

**DiscoveryRunStats table** (`discovery_run_stats`):
- id: Optional[int], primary key
- run_id: int, foreign_key="runs.id", indexed
- source_id: int, foreign_key="sources.id", indexed
- discovered_count: int, default 0
- matched_count: int, default 0
- error: Optional[str]
- created_at: datetime, default utcnow

Import these models in `app/db/models.py` at the bottom so Alembic's metadata picks them up. Add them to `__all__`.
  </action>
  <verify>

  </verify>

</task>
