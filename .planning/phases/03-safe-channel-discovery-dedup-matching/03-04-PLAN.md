  <name>Task 1: Create jobs router with sorting, detail, and queue endpoints</name>
  <files>app/web/routers/jobs.py, app/main.py</files>
  <action>
Create `app/web/routers/jobs.py` with APIRouter(prefix="/jobs").

Routes:

1. **GET /jobs** - Full jobs page or table body partial
   - Query params: `sort` (title|company|source|score|posted_date|status), `dir` (asc|desc), defaults sort=score, dir=desc
   - Load all jobs via `list_jobs(session, sort_by, sort_dir)`
   - Load Settings for match_threshold (needed for badge color logic)
   - Load keywords from Settings.keywords_csv (pipe-delimited) for passing to template
   - If HTMX request (check `HX-Request` header): return `partials/jobs_table.html.j2` (just tbody)
   - If full page request: return `jobs.html.j2` (full page with table)
   - Pass `threshold` to template for badge color computation

2. **GET /jobs/{job_id}/detail** - Inline expand partial
   - Load job by ID via `get_job_detail(session, job_id)`
   - Load keywords from Settings.keywords_csv
   - Parse job.matched_keywords (pipe-delimited) into list
   - Compute unmatched keywords = all_keywords - matched_keywords
   - Return `partials/job_detail_inline.html.j2`
   - Include: full description (HTML), matched keywords (green), unmatched keywords (gray), original URL link, "Queue for apply" button if status is "discovered" (below threshold)

3. **POST /jobs/{job_id}/queue** - Manual queue action
   - Call `update_job_status(session, job_id, "queued")`
   - Per user decision: "User can manually queue a below-threshold job from the expanded view"
   - Return updated detail partial (button changes to "Queued" disabled state)

Register in `app/main.py`: `from app.web.routers import jobs as jobs_router` and `app.include_router(jobs_router.router)`.

Add "Jobs" link to the base template navigation (next to Dashboard, Runs, Settings).

**jobs_table.html.j2:**
Table body rows (swapped by HTMX on sort):
- Each job row is clickable:
  ```
  <tr hx-get="/jobs/{{ job.id }}/detail" hx-target="#detail-{{ job.id }}" hx-swap="innerHTML" style="cursor: pointer;">
  ```
- Score badge with color coding per user decision:
  - Green: score >= threshold
  - Yellow: score >= threshold * 0.5 (borderline)
  - Gray: score < threshold * 0.5
  - Use CSS classes: `badge-green`, `badge-yellow`, `badge-gray`

- Status column shows status value
- Hidden detail row below each job: `<tr id="detail-{{ job.id }}" style="display:none;"><td colspan="6"></td></tr>`
- JavaScript-free expand: use HTMX's hx-on to toggle the detail row visibility. Alternatively, return the detail row with `style="display:table-row"` from the server and use hx-swap="outerHTML" on the hidden row.
