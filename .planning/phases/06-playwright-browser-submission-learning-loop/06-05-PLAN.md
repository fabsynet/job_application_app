  <files>app/playwright_submit/strategy.py, tests/playwright_submit/test_strategy.py</files>
  <action>
  Create `app/playwright_submit/strategy.py` with:

  **Helper: `_is_known_ats_url(url: str) -> bool`** — returns True if URL contains `greenhouse.io`, `lever.co`, or `ashbyhq.com`.

  **`class PlaywrightStrategy`:**
  - `name: str = "playwright"` — matches Submission.submitter column value
  - Constructor accepts `browser_manager: BrowserManager | None = None` and `session_factory = None`. If browser_manager is None, creates a lazy one on first use.
  - `def is_applicable(self, job: Job, description: str) -> bool` — returns True if `job.source in ("greenhouse", "lever", "ashby")` OR `_is_known_ats_url(job.url)`. Returns False otherwise (falls through to EmailStrategy).
  - `async def submit(self, ctx: SubmissionContext) -> SubmissionOutcome` — the main orchestration:

  **submit() flow:**
  1. Get or create BrowserManager. Read Settings for headless mode and pause_if_unsure toggle. Read data_dir from config.
  2. Get a new page from BrowserManager.
  3. Select the appropriate filler via `select_filler(ctx.job.source, ctx.job.url)`.
  4. Navigate to the form URL (via filler's navigate_to_form).
  5. Check for CAPTCHA/2FA via `detect_blocking_element(page)`. If found: screenshot error, return `SubmissionOutcome(success=False, submitter="playwright", error_class="captcha", error_message=blocking_type)`. The pipeline (05-04) will handle the status transition.
  6. Load Profile from DB (lazily via session_factory or import get_profile_row).
  7. Load all SavedAnswers from DB.
  8. Scan all form pages via filler's `scan_all_pages()`. Collect known_fields + unknown_fields. Screenshot each step.
  9. For unknown fields: call `try_match_and_fill()` from the learning matcher to check against saved answers. Split into auto-filled (matched) and truly-unknown.
  10. If truly-unknown fields exist AND pause_if_unsure is True:
      - Persist unknown fields to DB via `create_unknown_fields()`
      - Return `SubmissionOutcome(success=False, submitter="playwright", error_class="needs_info", error_message=f"{len(truly_unknown)} unknown fields")`
  11. If pause_if_unsure is False (always-advance mode): skip unknowns, proceed to fill and submit.
  12. Fill all known fields + auto-matched fields via filler's `fill_and_submit()`.
  13. Wait for submission confirmation via filler's `detect_success()`.
  14. Save storageState via BrowserManager.
  15. Capture final screenshot (confirmation page).
  16. Return `SubmissionOutcome(success=True, submitter="playwright")`

  **Error handling in submit():**
  - Wrap entire flow in try/except. On any PlaywrightError, TimeoutError, or unexpected exception:
    - Take error screenshot via `capture_error_screenshot()`
    - Return `SubmissionOutcome(success=False, submitter="playwright", error_class=type(exc).__name__, error_message=str(exc))`
  - Never raise — always return an outcome (fail-fast pattern per locked decision)

  **`async def close(self)` — close the BrowserManager if it was created internally.**

  **`reused_answers` property** — list of (field_label, saved_answer) pairs that were auto-filled via the matcher. The pipeline (or notification sender) can read this to include in the success email per locked decision.

  - test_submit_unknown_fields_advance: unknown fields + pause_if_unsure=False -> proceeds to submit
  - test_submit_error_screenshot: Playwright error -> takes error screenshot + returns failure outcome
  - test_submit_auto_match: unknown field matched via LLM -> auto-filled, not halted
