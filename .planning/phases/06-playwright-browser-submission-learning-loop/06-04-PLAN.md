  <name>Task 2: LLM semantic matcher + needs-info aggregation</name>
  <files>app/learning/matcher.py, app/learning/needs_info.py, tests/learning/test_matcher.py, tests/learning/test_needs_info.py</files>
  <action>
  **app/learning/matcher.py:**

  `async def find_matching_answers(field_labels: list[str], saved_answers: list[SavedAnswer], provider) -> dict[str, SavedAnswer | None]`:
  - If no saved_answers, return {label: None for label in field_labels}
  - Batch ALL field labels into ONE LLM call per form (per research recommendation on cost)
  - Build prompt:
    ```
    Given these new form field labels:
    {numbered list of field_labels}

    And these previously answered field labels with IDs:
    {numbered list of saved_answer field_labels with their ids}

    For each new label, return the ID of the matching previously answered label if they ask the same question (even with different wording), or "NONE" if no match.

    Return as JSON: {"matches": [{"new_label": "...", "saved_id": N or null}, ...]}
    ```
  - Use `provider.complete(system=[...], messages=[...], max_tokens=500, temperature=0.0)`
  - Parse the JSON response. For each match with a non-null saved_id, look up the SavedAnswer
  - Per locked decision: **always auto-fill if LLM says match** — no confidence threshold
  - Return dict mapping each field_label to its matching SavedAnswer or None
  - If LLM call fails (budget, network), return all None (graceful degradation — fields become unknown)

  `async def try_match_and_fill(session, unknown_field_infos: list, saved_answers: list[SavedAnswer], provider) -> tuple[list, list]`:
  - Call find_matching_answers for the unknown field labels
  - Split into: matched (have a SavedAnswer) and still_unknown (no match)
  - For each matched: increment reuse count on the SavedAnswer
  - Return (matched_with_answers, still_unknown)

  **app/learning/needs_info.py:**

  `async def get_needs_info_jobs(session) -> list[dict]`:
  - Query jobs with status='needs_info'
  - For each job, load its unresolved unknown_fields
  - Return list of dicts: `{job: Job, unknown_fields: list[UnknownField], field_count: int}`
  - Ordered by most recent (job.id DESC)

  `async def get_needs_info_detail(session, job_id: int) -> dict | None`:
  - Load job + all unknown_fields (resolved and unresolved) for that job
  - Return `{job: Job, unknown_fields: list[UnknownField], resolved_count: int, total_count: int}` or None

  Tests:
  - test_matcher.py: Use a FakeLLMProvider (local test class, same pattern as 04-07) that returns scripted JSON responses. Test: two field labels matched, one unmatched. Test: empty saved_answers returns all None. Test: LLM failure returns all None. Test: reuse count incremented on match.
  - test_needs_info.py: Use async_session. Create jobs with status='needs_info' and UnknownField rows. Test: get_needs_info_jobs returns correct aggregation. Test: get_needs_info_detail returns resolved/total counts. Test: resolved fields excluded from unresolved count.
  </action>
