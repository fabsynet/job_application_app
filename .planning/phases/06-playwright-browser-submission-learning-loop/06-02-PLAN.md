  Create `app/playwright_submit/browser.py` with a `BrowserManager` class:

  ```python
  class BrowserManager:
      """Manages a single Playwright browser instance per pipeline run (SUBM-05)."""
  ```

  Key design:
  - Constructor accepts `headless: bool = True` and `storage_state_dir: Path | None = None` (defaults to `data/browser/`)
  - `storage_state_path` property returns `{storage_state_dir}/storageState.json`
  - `async get_context()` — lazily starts playwright + launches chromium + creates a context. Loads storageState from file if it exists. Sets default timeouts: action 30_000ms, navigation 60_000ms.
  - `async save_state()` — calls `context.storage_state(path=self.storage_state_path)`. Creates parent dir if missing.
  - `async close()` — closes context, browser, stops playwright. Idempotent (no error if already closed or never started).
  - Internal state: `_playwright`, `_browser`, `_context` all initially None
  - `async get_page()` — convenience: calls `get_context()` then returns `context.new_page()`
  - Context manager support: `async __aenter__` returns self, `async __aexit__` calls close()

  Tests in `tests/playwright_submit/test_browser.py`:
  - Mock `async_playwright` using `unittest.mock.AsyncMock` / `MagicMock` so no real browser is launched
  - Test: get_context creates playwright + browser + context with correct headless setting
  - Test: get_context loads storageState when file exists (mock Path.exists)
  - Test: get_context skips storageState when file doesn't exist
  - Test: save_state writes storageState JSON (mock context.storage_state)
  - Test: close is idempotent — calling twice doesn't error
  - Test: context manager support (__aenter__/__aexit__)
  </action>
  <verify>Run `python -m pytest tests/playwright_submit/test_browser.py -v`</verify>
  <done>BrowserManager launches Playwright with configurable headless/storageState, saves state after submissions, and cleans up on close. All tests pass with mocked Playwright.</done>

  **app/playwright_submit/captcha.py:**

  Create `async def detect_blocking_element(page) -> str | None`:
  - Check selectors in priority order (per research discretionary recommendation):
    1. `iframe[src*="recaptcha"]` or `div.g-recaptcha` -> "recaptcha"
    2. `iframe[src*="hcaptcha"]` or `div.h-captcha` -> "hcaptcha"
    3. `iframe[src*="challenges.cloudflare.com"]` or `#challenge-running` -> "cloudflare"
    4. `[class*="captcha" i]` or `[id*="captcha" i]` -> "generic_captcha"
    5. `input[name*="otp"]` or `input[name*="verification_code"]` or `[class*="two-factor"]` -> "2fa"
  - For each selector, use `page.locator(selector).count()` — if > 0, return the label
  - Also check for login redirect: if current URL contains "/login", "/signin", "/sso" -> "login_required"
  - Return None if no blocking element found

  **app/playwright_submit/screenshots.py:**

  Create utilities:
  - `def screenshot_dir(data_dir: Path, job_id: int) -> Path` — returns `{data_dir}/screenshots/{job_id}/`, creates it if missing
  - `async def capture_step_screenshot(page, data_dir: Path, job_id: int, step: int) -> str` — saves to `screenshots/{job_id}/step_{step}.png`, returns the relative path from data_dir
  - `async def capture_error_screenshot(page, data_dir: Path, job_id: int) -> str` — saves to `screenshots/{job_id}/error.png`, returns the relative path
  - `def cleanup_old_screenshots(data_dir: Path, retention_days: int, exempt_job_ids: set[int] | None = None) -> int` — deletes screenshot dirs older than retention_days, skips exempt jobs (needs_info), returns count deleted. Use directory mtime for age check.
