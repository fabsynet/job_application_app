  **LABEL_HEURISTICS** — a list of tuples `(pattern: str, profile_field: str, input_method: str)` in priority order (per research discretionary recommendation):
  ```python
  LABEL_HEURISTICS = [
      (r"\bfirst\s*name\b", "first_name", "fill"),
      (r"\blast\s*name\b", "last_name", "fill"),
      (r"\bfull\s*name\b", "full_name", "fill"),
      (r"\bname\b", "full_name", "fill"),  # fallback after first/last
      (r"\bemail\b", "email", "fill"),
      (r"\bphone\b", "phone", "fill"),
      (r"\blinkedin\b", "linkedin_url", "fill"),
      (r"\bgithub\b", "github_url", "fill"),
      (r"\bportfolio\b|personal\s*(?:site|website|url)\b", "portfolio_url", "fill"),
      (r"\bresume\b|\bcv\b", "_resume_file", "upload"),
      (r"\bcover\s*letter\b", "_cover_letter_file", "upload"),
      (r"\bwork\s*auth", "work_authorization", "select_or_fill"),
      (r"\blegally\s*auth", "work_authorization", "select_or_fill"),
      (r"\bsponsorship\b", "work_authorization", "select_or_fill"),
      (r"\bsalary\b|\bcompensation\b", "salary_expectation", "fill"),
      (r"\byears?\s*(?:of\s*)?exp", "years_experience", "fill"),
      (r"\blocation\b|\bcity\b|\baddress\b|\bzip\b", "address", "fill"),
  ]
  ```

  **`match_field_to_profile(label: str) -> tuple[str, str] | None`** — iterate LABEL_HEURISTICS, return `(profile_field, input_method)` for first case-insensitive regex match, or None.

  **`get_profile_value(profile: Profile, field_name: str) -> str | None`** — resolve profile_field to a value. Handle special cases:
  - `first_name` -> `profile.full_name.split()[0]` if full_name exists
  - `last_name` -> `profile.full_name.split()[-1]` if full_name has 2+ words
  - `full_name` -> `profile.full_name`
  - All other fields -> `getattr(profile, field_name, None)`, convert to str if not None

  **Dataclasses:**
  ```python
  @dataclass
  class KnownField:
      label: str
      profile_field: str
      value: str
      input_method: str  # fill | upload | select_or_fill
      locator: Any  # Playwright Locator

  @dataclass
  class UnknownFieldInfo:
      label: str
      field_type: str  # text | select | checkbox | file | radio
      options: list[str] | None  # for select/radio
      is_required: bool
      page_number: int
      locator: Any  # Playwright Locator
  ```

  **`async def classify_fields(page, profile: Profile, page_number: int = 1) -> tuple[list[KnownField], list[UnknownFieldInfo]]`:**
  - Find all visible form fields on the page: `input:visible`, `select:visible`, `textarea:visible` (excluding hidden and submit/button types)
  - For each field, extract the label text using:
    1. Associated `<label>` element via `for` attribute
    2. Closest `<label>` ancestor
    3. `aria-label` attribute
    4. `placeholder` attribute
  - Call `match_field_to_profile(label)` to classify
  - If matched AND profile has a value: create KnownField
  - If unmatched or profile value is None: create UnknownFieldInfo with detected field_type (inspect tag name + input type attribute), options (for select: extract all option texts), is_required (check `required` attr or `aria-required`)
  - Skip file upload fields that are unknown per locked decision (skip, don't halt), UNLESS they have `required` attribute — then add as UnknownFieldInfo
  - Return (known_fields, unknown_fields)

  **`async def fill_known_fields(page, known_fields: list[KnownField], resume_path: Path | None, cover_letter_path: Path | None) -> None`:**
  - For each known field by input_method:
    - "fill": `field.locator.fill(field.value)`
    - "upload": `field.locator.set_input_files(resume_path or cover_letter_path)` based on profile_field
    - "select_or_fill": Try `field.locator.select_option(label=field.value)` first. If that fails (no exact match), try case-insensitive substring match against options. If still no match, try `field.locator.fill(field.value)` as fallback for text-input work-auth fields.
  - Handle checkbox/radio for boolean fields: `field.locator.check()`

  **`async def try_select_with_fallback(locator, value: str) -> bool`:**
  - Exact match first: `select_option(label=value)`
  - Case-insensitive substring: iterate options, find one containing `value.lower()`
  - Return True if matched, False if not (caller treats as unknown per dropdown locked decision)

  Create `app/playwright_submit/fillers/__init__.py` with a `select_filler(job_source: str, job_url: str)` function that returns the appropriate filler class.

  Each filler class follows the same interface:
  ```python
  class XxxFiller:
      async def get_form_url(self, job: Job) -> str
      async def navigate_to_form(self, page) -> None
      async def scan_all_pages(self, page, profile, resume_path, cover_letter_path) -> tuple[list[KnownField], list[UnknownFieldInfo]]
      async def fill_and_submit(self, page, known_fields, resume_path, cover_letter_path) -> None
      async def detect_success(self, page) -> bool
  ```

  **GreenhouseFiller:**
  - `get_form_url`: construct `https://boards.greenhouse.io/{slug}/jobs/{external_id}#app` from job.company (slug) + job.external_id. If job.url contains `boards.greenhouse.io`, use it directly.
  - `navigate_to_form`: `page.goto(url)`, wait for networkidle. Check for iframe embedding — if form isn't found directly, try `page.frame_locator("iframe[src*='boards.greenhouse.io']")`.
  - `scan_all_pages`: Greenhouse forms are typically single-page. Call `classify_fields`. Check for "Additional Information" sections with custom questions. Handle GDPR consent checkboxes (auto-check if label contains "consent" or "agree" and field is a checkbox).
  - `fill_and_submit`: fill known fields, click submit button matching `input[type="submit"][value*="Submit"], button:has-text("Submit Application")`
  - `detect_success`: check for confirmation text ("Thank you", "Application submitted", "received your application")

  **LeverFiller:**
  - `get_form_url`: construct `https://jobs.lever.co/{slug}/{external_id}/apply`
  - `navigate_to_form`: `page.goto(url)`, wait for networkidle.
  - `scan_all_pages`: Lever forms are typically single-page. Standard fields: name, email, phone, org (current company), URLs section (LinkedIn, GitHub, etc.), resume upload.
  - `fill_and_submit`: fill known, click `button[type="submit"]:has-text("Submit"), button:has-text("Submit application")`
  - `detect_success`: check for redirect or confirmation message

  **AshbyFiller:**
  - `get_form_url`: construct `https://jobs.ashbyhq.com/{slug}/{external_id}/application`
  - `navigate_to_form`: `page.goto(url)`, wait for networkidle. Ashby uses JS embed — wait for form to render.
  - `scan_all_pages`: Ashby can have multi-step forms. Iterate through pages using "Next"/"Continue" buttons. Call `classify_fields` on each page. Accumulate known + unknown across all pages.
  - `fill_and_submit`: Navigate back to first page, fill all known fields page by page, then submit on the final page.
  - `detect_success`: check for confirmation/thank you page

  **GenericFiller:**
  - `navigate_to_form`: `page.goto(job.url)`, wait for networkidle. Use progressive form detection (per research discretionary recommendation):
    1. Look for `<form>` containing file upload inputs
    2. Look for `[role="form"]` or `[aria-label*="application"]` or `[aria-label*="apply"]`
    3. Look for visible `<form>` with 3+ input fields
    4. Prefer form with file input (resume upload signals application form)
    5. Fall back to largest visible form
  - `scan_all_pages`: similar to Ashby — check for multi-step, accumulate all fields
  - `fill_and_submit`: fill known, look for submit buttons with priority: `button:has-text("Submit"), button:has-text("Apply"), input[type="submit"]`
  - `detect_success`: generic check for confirmation keywords

  **`select_filler(job_source, job_url)` logic:**
  - `greenhouse` or URL contains `greenhouse.io` -> GreenhouseFiller()
  - `lever` or URL contains `lever.co` -> LeverFiller()
  - `ashby` or URL contains `ashbyhq.com` -> AshbyFiller()
  - else -> GenericFiller()

  - test_gdpr_checkbox_autocheck: Greenhouse consent checkbox auto-filled
