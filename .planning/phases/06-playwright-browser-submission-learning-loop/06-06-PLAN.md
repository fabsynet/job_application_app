  <files>app/web/routers/needs_info.py, app/web/templates/needs_info/index.html.j2, app/web/templates/needs_info/detail.html.j2, app/web/templates/needs_info/answer_form.html.j2, tests/needs_info/__init__.py, tests/needs_info/test_needs_info_router.py</files>
  <action>
  Create `app/web/routers/needs_info.py` with an APIRouter(prefix="/needs-info"):

  **GET /needs-info** — Queue listing page
  - Call `get_needs_info_jobs(session)` from `app.learning.needs_info`
  - Render `needs_info/index.html.j2` with the list of jobs + their unknown field counts
  - Each row shows: job title, company, source, field count, timestamp
  - Clickable rows link to the detail page

  **GET /needs-info/{job_id}** — Job detail with unknown fields
  - Call `get_needs_info_detail(session, job_id)` from `app.learning.needs_info`
  - 404 if job not found or not in needs_info status
  - Render `needs_info/detail.html.j2` with:
    - Job context: title, company, source, description (full), match score
    - Each unknown field: screenshot (if available, rendered as `<img>`), field label, field type, options (for select/radio), is_required badge, page number
    - Answer form for all unresolved fields (one form with all fields, submit button at bottom)

  **POST /needs-info/{job_id}/answer** — Submit answers for all unknown fields
  - Parse form data: field_{unknown_field_id} = answer_text for each field
  - Call `resolve_all_for_job(session, job_id, answers_dict)` where answers_dict maps field_id -> answer_text
  - After resolving, flip job status from `needs_info` to `approved` via `flip_job_status(session, job_id, "approved", reason="user_answered_unknowns")`
  - Redirect to the retry endpoint (or render a "Retrying..." page)

  **POST /needs-info/{job_id}/retry** — Immediate Playwright retry (LEARN-05)
  - Verify job status is `approved` (just answered) or allow re-retry
  - Create a PlaywrightStrategy instance with a BrowserManager
  - Build a minimal SubmissionContext (job, tailored_resume_path from TailoringRecord, cover_letter_path)
  - Call `strategy.submit(ctx)` inline (not via scheduler — instant feedback per locked decision)
  - On success: flip job to `submitted`, create Submission row, redirect to needs-info index with success toast
  - On failure: flip job to `failed` (or back to `needs_info` if new unknowns), redirect with error toast
  - Close the BrowserManager after

  Templates:
  - `needs_info/index.html.j2`: extends base.html.j2. Table with columns: Job, Company, Source, Unknown Fields, Date. Empty state: "No halted applications." Link back to dashboard.
  - `needs_info/detail.html.j2`: extends base.html.j2. Two-column layout: left = job context (title, company, description), right = unknown fields with answer inputs. Screenshots shown inline. Submit button: "Save Answers & Retry Now"
  - `needs_info/answer_form.html.j2`: partial for HTMX swap if needed. Contains the form fields: for text fields -> `<input>`, for select fields -> `<select>` with options, for checkbox -> `<input type="checkbox">`. Each field labeled with the original field_label + screenshot thumbnail.

  Style with Pico.css classes (matching existing app aesthetic). Use HTMX for the retry button (hx-post, show loading indicator).

  Tests:
  - test_needs_info_list: create jobs in needs_info status + unknown fields, GET /needs-info returns 200 with job entries
  - test_needs_info_list_empty: no needs_info jobs -> 200 with empty state message
  - test_needs_info_detail: GET /needs-info/{id} returns 200 with unknown fields
  - test_needs_info_detail_404: wrong job_id or non-needs_info job -> 404
  - test_answer_and_resolve: POST /needs-info/{id}/answer with form data -> fields resolved, job flipped to approved
  - test_answer_creates_saved_answers: verify SavedAnswer rows created after answering
  </action>
